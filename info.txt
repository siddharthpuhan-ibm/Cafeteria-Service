Typical Workflow

When you start a new feature, create a feature branch: git checkout -b feature-auth

Do work → commit → push:

git add .
git commit -m "Add mock auth"
git push origin feature-auth

When finished, merge after review:

git checkout main
git merge feature-auth
git push


check current branch: git branch
check git status: git status

git only pushes commits, not edited files. So, if a file is edited then it will be shown as modified after running "git status"
if you see modifed files then create a commit and push

git add .
git commit -m "your message"
git push

Run the uvicorn server: uvicorn app.main:app --reload
